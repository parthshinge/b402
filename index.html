How It Works

1

Request

A buyer (agent) requests a resource from a b402-enabled service.

2

Payment Required

Service responds with 402 Payment Required and b402-formatted payment instructions.

3

Payment

Buyer sends payment payload to the b402 facilitator for settlement.

4

Verification

Server verifies the transaction through /verify and /settle endpoints.

5

Access Granted

Once verified, the server releases the requested resource.

Frequently Asked Questions

What is b402?

b402 is an open payment protocol standard on BNB Chain that enables seamless agent-to-agent payments with native crypto primitives.

How does b402 handle identity?

b402 uses ERC-8004 for verified agent transactions, ensuring secure and authenticated interactions.

What are the transaction fees?

Transaction fees are minimal, leveraging BNB Chain's low-cost infrastructure while maintaining security and speed.

Is b402 compatible with other blockchains?

While initially deployed on BNB Chain, b402 is designed with EVM compatibility in mind for future expansion.
